# aiscrub - Take credit for AI work

Find and remove **AI attributions** from a git repo — both commit messages
across every ref, and tracked files in the working tree. Optionally do the
opposite: ironically attribute every commit to Claude (or any text you choose).

Covers Claude, GitHub Copilot, Cursor, ChatGPT/OpenAI, Aider, Sourcegraph Cody,
Codeium/Windsurf, Devin, Gemini/Jules, Tabnine, JetBrains AI, Continue.dev, and
generic "generated by an AI" phrasings.

> ⚠️ `scrub` and `dirty` rewrite git history. They change commit SHAs, break
> open PRs against the old SHAs, invalidate signed commits, and force every
> collaborator to re-clone or hard-reset. Read the warnings before running.

## Install

### From a release (Windows)

Releases ship a **Windows x86_64** binary only (the project is developed on
Windows, so that's the only OS we can build a binary for in CI). Download
`aiscrub-windows-x86_64.zip` from the [Releases](../../releases) page, unzip,
and put `aiscrub.exe` on your `PATH`.

Verify with `SHA256SUMS.txt` from the same release.

### From source (any OS)

The Python source is cross-platform. Linux and macOS users build the binary
themselves:

```bash
git clone https://github.com/robot-pizza/aiscrub
cd aiscrub
uv sync --extra build
uv run aiscrub --help                                       # run from source
uv run pyinstaller --onefile --name aiscrub aiscrub.py      # build a binary
```

## Usage

```text
aiscrub scan
    Read-only report. Lists every commit and tracked file containing an
    AI attribution. Exits non-zero if any are found (CI-friendly).

aiscrub scrub
    DRY-RUN by default. Shows what would change. Safe.

aiscrub scrub --not-dry-run
    DESTRUCTIVE. Rewrites history and tracked files. Prompts first.

aiscrub scrub --not-dry-run --yes
    DESTRUCTIVE and unattended. No prompt.

aiscrub dirty
    DRY-RUN preview of the inverse: ADDING an attribution trailer to every
    commit that does not already have one. Idempotent.

aiscrub dirty --not-dry-run
    DESTRUCTIVE inverse. Appends an attribution trailer to every
    un-attributed commit on every ref.

aiscrub dirty --attribution "..."
aiscrub dirty --attribution-file ./trailer.txt
    Use a custom trailer block instead of the default Claude one.
```

`scrub` does two things, in order:

1. Strips attribution lines from every tracked text file and commits the result.
2. Rewrites every commit message on every ref using `git filter-repo` (falling
   back to `git filter-branch` if filter-repo is not installed).

After a real `scrub` or `dirty`, the tool prints the exact
`git push --force-with-lease` commands to publish, plus a step-by-step playbook
for temporarily relaxing GitHub branch protection for yourself only — and
reverting it when done.

## What gets matched

Patterns are intentionally forgiving (unicode dashes, NBSP, optional brackets,
alternate phrasings). Coverage includes:

- `Co-Authored-By: Claude <…>` and `@anthropic.com` trailers
- `Co-Authored-By: copilot-swe-agent` / `github-copilot`, "Generated with GitHub Copilot"
- Cursor, ChatGPT/OpenAI/Codex, Aider, Cody, Codeium/Windsurf, Devin, Gemini/Jules,
  Tabnine, Continue.dev, JetBrains AI Assistant
- Bracketed/linkified `[Claude Code]` references and `claude.com/claude-code`,
  `claude.ai/code` URLs
- Lines led by 🤖 / `:robot:` that mention Claude
- Generic "Generated by AI / LLM / large language model" catch-all

## Force-push playbook (GitHub)

If your target branch is protected, the force push will be rejected. The full
flow (web UI + `gh` CLI versions) is printed after a real run, but the short
version:

1. Repo Settings → Branches (or Rules) → temporarily allow force-push **for
   your user only**.
2. `git push --force-with-lease --all origin && git push --force-with-lease --tags origin`
3. Revert the branch protection change.
4. Tell collaborators to `git fetch && git reset --hard origin/<branch>` or reclone.

## Build a binary locally

```bash
make build      # produces dist/aiscrub (or dist\aiscrub.exe)
make zip        # produces aiscrub-<os>-<arch>.zip
make release    # tags v$VERSION and pushes (triggers GitHub release workflow)
```

Or directly:

```bash
uv sync --extra build
uv run pyinstaller --onefile --name aiscrub aiscrub.py
```

## Commit convention for this repo (ironic)

This project strips AI attributions from other people's repos. Commits to
**this** repo deliberately keep the attribution, with a winky emoji:

```
🤖 Generated with [Claude Code](https://claude.com/claude-code) 😉

Co-Authored-By: Claude <noreply@anthropic.com>
```

Do not remove it. See `CONTRIBUTING.md`.
